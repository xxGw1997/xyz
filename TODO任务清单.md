# AI Chat Agent TODO 任务清单

目标：把当前项目补成一个更完整的 AI Chat Agent，优先保证“能聊、会调工具、能记上下文、能接知识库”，个性化能力放后面。

说明：你提到的“Agent 基础体验”和“产品化能力”这两块目前先不放进 TODO，默认认为已经具备或正在可用状态。

## 一、核心 Agent 能力

- [x] 支持 tool-call 机制
- [ ] 区分客户端 tool 和服务端 tool
- [x] 明确哪些 tool 需要用户同意后才能执行
- [x] 支持 tool 执行结果回传到对话上下文
- [ ] 处理 tool 调用失败、超时、重试和异常提示
- [ ] 支持不同场景选择不同模型
- [ ] 支持多模型切换

## 二、上下文与成本控制

- [ ] 优化历史消息构建逻辑
- [ ] 当上下文过长时自动裁剪
- [ ] 对旧消息做摘要压缩，减少 token 消耗
- [ ] 为不同会话设置上下文长度上限
- [ ] 保留关键记忆，避免裁剪后丢失核心信息
- [ ] 记录 token 使用量和调用成本
- [ ] 对长对话做自动总结

## 三、会话管理

- [ ] 根据首条消息自动生成会话标题
- [ ] 支持手动修改会话标题
- [ ] 支持删除会话
- [ ] 支持会话置顶 / 收藏
- [ ] 支持会话搜索与筛选

## 四、知识库与检索

- [ ] 新增“日记”或“资料”知识库管理
- [ ] 管理员可上传文本资料
- [ ] 支持资料分段、清洗和入库
- [ ] 生成向量索引用于检索
- [ ] 对话时支持基于 RAG 检索知识库回答
- [ ] 支持知识库的增删改查
- [ ] 支持知识库权限控制
- [ ] 对 RAG、tool-call、上下文构建做性能优化

## 五、个性化能力

- [ ] 新增人格创建功能
- [ ] 新增人格编辑功能
- [ ] 新增人格删除功能
- [ ] 支持为人格配置系统提示词
- [ ] 对话时可切换人格
- [ ] 将人格提示词作为 LLM system prompt 注入
- [ ] 支持默认人格和常用人格

## 六、扩展能力

- [ ] 支持图片上传和图片理解
- [ ] 支持文件上传和文件解析
- [ ] 支持语音输入
- [ ] 支持语音播报
- [ ] 支持联网搜索能力
- [ ] 支持更多外部工具接入

## 七、可继续补充

- [ ] 记忆功能：保存用户偏好和长期信息
- [ ] Agent 计划能力：先分析再执行
- [ ] 多轮任务拆解与任务状态管理
- [ ] Prompt 模板管理
- [ ] 对话评价与反馈系统
- [ ] 导出聊天记录
- [ ] 分享单个会话链接

## 建议优先级

1. 先做核心 Agent 能力和上下文控制
2. 再做会话管理和知识库 RAG
3. 然后补个性化能力
4. 最后做多模态、记忆和其他扩展能力
